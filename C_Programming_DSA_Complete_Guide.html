Introduction to C & Structure of a C Program

About C Language

C is a general-purpose, procedural, middle-level programming language developed by Dennis Ritchie in 1972 at Bell Labs. It is the foundation of many modern languages (C++, Java, Python are influenced by C).

  • Fast & Efficient — close to hardware, minimal runtime overhead
  • Portable — code runs on different machines with little change
  • Structured — supports functions, loops, and conditional logic
  • Rich Library — built-in standard library functions
  • Used in OS development (UNIX/Linux written in C), embedded systems, compilers

Structure of a C Program

/* 1. Documentation / Comment Section */
/* Program Name: Hello World  Author: Student */

/* 2. Preprocessor / Header File Section */
#include <stdio.h>    // Standard Input Output
#include <conio.h>   // Console I/O (optional)

/* 3. Global Declaration Section */
int globalVar = 10;

/* 4. Main Function — Entry Point */
int main() {

    /* 5. Local Variable Declaration */
    int a = 5;
    float b = 3.14;

    /* 6. Executable Statements */
    printf("Hello, World!\n");
    printf("a = %d, b = %.2f\n", a, b);

    return 0;   // Return to OS
}

/* 7. User-Defined Functions (if any) */
void myFunction() {
    printf("User defined function\n");
}
Execution Flow: Preprocessor → Compiler → Assembler → Linker → Executable

Define an Algorithm

An Algorithm is a finite, step-by-step, well-defined sequence of instructions to solve a specific problem or perform a computation.

Properties of a Good Algorithm

  • Input — Takes zero or more inputs
  • Output — Produces at least one output
  • Definiteness — Each step must be clear and unambiguous
  • Finiteness — Must terminate after a finite number of steps
  • Effectiveness — Each step must be feasible/executable

Example: Algorithm to find sum of two numbers

Step 1: START
Step 2: Read two numbers A and B
Step 3: SUM = A + B
Step 4: Print SUM
Step 5: STOP

Conditional Statements — Nested if-else vs else-if Ladder

Conditional statements allow decision making in a program — different code executes based on conditions.

Types

  • if — single condition
  • if-else — two-way decision
  • nested if-else — if-else inside another if-else
  • else-if ladder — multiple conditions in sequence

Nested if-else

if (condition1) {
    if (condition2) {
        // Both true
    } else {
        // condition1 true, condition2 false
    }
} else {
    // condition1 false
}
// Example: Largest of 3 numbers
if (a > b) {
    if (a > c) printf("a is largest");
    else        printf("c is largest");
} else {
    if (b > c) printf("b is largest");
    else        printf("c is largest");
}

else-if Ladder

if (marks >= 90)        printf("Grade A");
else if (marks >= 75)   printf("Grade B");
else if (marks >= 60)   printf("Grade C");
else if (marks >= 40)   printf("Grade D");
else                     printf("Fail");

Loops — while vs do-while

A loop is a control structure that repeats a block of code as long as a condition is true. Loops avoid writing repetitive code.

Types of Loops in C

  • for loop — when number of iterations is known
  • while loop — entry-controlled, condition checked before execution
  • do-while loop — exit-controlled, condition checked after execution

while Loop

// Syntax
while (condition) {
    // body
}

// Example: Print 1 to 5
int i = 1;
while (i <= 5) {
    printf("%d ", i);
    i++;
}
// Output: 1 2 3 4 5

do-while Loop

// Syntax
do {
    // body (executes at least once)
} while (condition);

// Example
int i = 1;
do {
    printf("%d ", i);
    i++;
} while (i <= 5);
// Output: 1 2 3 4 5

Difference Table

Featurewhiledo-while
TypeEntry-controlledExit-controlled
Condition checkBefore loop bodyAfter loop body
Minimum executions0 (if false initially)1 (always executes once)
Use caseWhen iterations may be zeroMenu-driven programs, input validation
SemicolonNot required after while()Required after while(condition);

Switch Statement in C

The switch statement is a multi-way decision statement. It tests the value of an expression against a list of integer or character constants and executes the matching case.

Syntax & Example

int day;
printf("Enter day number (1-7): ");
scanf("%d", &day);

switch (day) {
    case 1:
        printf("Monday\n");
        break;
    case 2:
        printf("Tuesday\n");
        break;
    case 3:
        printf("Wednesday\n");
        break;
    case 6:
    case 7:
        printf("Weekend!\n");
        break;
    default:
        printf("Invalid day\n");
}
Important Rules:
• The expression in switch must be int or char (not float/string)
• break exits the switch — without it, execution "falls through" to next case
• default is optional but handles unmatched values
• Two cases cannot have the same constant value
Q 7

Short Notes: break, continue, goto

1. break

Terminates the nearest enclosing loop or switch and transfers control to the statement after it.

for (int i = 1; i <= 10; i++) {
    if (i == 5) break;   // stops at 5
    printf("%d ", i);
}
// Output: 1 2 3 4

2. continue

Skips the rest of the current iteration and jumps to the next iteration of the loop (does NOT exit the loop).

for (int i = 1; i <= 6; i++) {
    if (i == 3) continue;  // skip 3
    printf("%d ", i);
}
// Output: 1 2 4 5 6

3. goto

Transfers control unconditionally to a labeled statement anywhere in the same function. Generally discouraged as it makes code hard to read.

int i = 1;
loop:
    if (i <= 5) {
        printf("%d ", i);
        i++;
        goto loop;
    }
// Output: 1 2 3 4 5
KeywordEffectUsed In
breakExit loop/switch immediatelyloops, switch
continueSkip current iterationloops only
gotoJump to a labelanywhere in function
Q 8

Program: Check Prime Number

A prime number is a number greater than 1 that has no divisors other than 1 and itself.

#include <stdio.h>
#include <math.h>

int main() {
    int n, i, isPrime = 1;
    printf("Enter a number: ");
    scanf("%d", &n);

    if (n <= 1) {
        isPrime = 0;
    } else {
        // Check divisors up to sqrt(n) — efficient
        for (i = 2; i <= (int)sqrt(n); i++) {
            if (n % i == 0) {
                isPrime = 0;
                break;
            }
        }
    }

    if (isPrime)
        printf("%d is a PRIME number.\n", n);
    else
        printf("%d is NOT a prime number.\n", n);

    return 0;
}
Logic: We check divisors only up to √n. If any number divides n evenly, it's not prime. Compile with: gcc prime.c -lm -o prime (for math.h)

Program: Fibonacci Series

Fibonacci series: 0, 1, 1, 2, 3, 5, 8, 13, 21 ... — each term = sum of previous two.

#include <stdio.h>

int main() {
    int n, a = 0, b = 1, next;
    printf("Enter number of terms: ");
    scanf("%d", &n);

    printf("Fibonacci Series: ");
    for (int i = 1; i <= n; i++) {
        printf("%d ", a);
        next = a + b;
        a = b;
        b = next;
    }
    printf("\n");
    return 0;
}
// Input: 8  → Output: 0 1 1 2 3 5 8 13

Q 11

Program: 2D Matrix Multiplication (2×2)

#include <stdio.h>

int main() {
    int A[2][2], B[2][2], C[2][2];
    int i, j, k;

    printf("Enter Matrix A (2x2):\n");
    for(i=0; i<2; i++)
        for(j=0; j<2; j++)
            scanf("%d", &A[i][j]);

    printf("Enter Matrix B (2x2):\n");
    for(i=0; i<2; i++)
        for(j=0; j<2; j++)
            scanf("%d", &B[i][j]);

    // Matrix Multiplication: C[i][j] = sum of A[i][k]*B[k][j]
    for(i=0; i<2; i++) {
        for(j=0; j<2; j++) {
            C[i][j] = 0;
            for(k=0; k<2; k++)
                C[i][j] += A[i][k] * B[k][j];
        }
    }

    printf("Result Matrix C:\n");
    for(i=0; i<2; i++) {
        for(j=0; j<2; j++)
            printf("%d\t", C[i][j]);
        printf("\n");
    }
    return 0;
}

Call by Value vs Call by Reference

FeatureCall by ValueCall by Reference
What is passedCopy of the valueAddress (pointer) of the variable
Original valueNOT modifiedIS modified
MemoryNew memory for copySame memory location used
SafetySafe — original protectedLess safe — original can change
Use caseWhen original must not changeWhen result must be reflected back
// Call by Value
void changeVal(int x) { x = 100; }  // local copy changed
int a = 10;
changeVal(a);
printf("%d", a);  // Still 10

// Call by Reference
void changeRef(int *x) { *x = 100; }  // original changed via pointer
int b = 10;
changeRef(&b);
printf("%d", b);  // Now 100

// Classic swap using Call by Reference
void swap(int *p, int *q) {
    int temp = *p;
    *p = *q;
    *q = temp;
}

Q 14

Pointers in C

A pointer is a variable that stores the memory address of another variable.

int  a  = 10;         // normal variable
int *p  = &a;         // pointer p stores address of a

printf("Value of a:   %d\n", a);    // 10
printf("Address of a: %p\n", &a);   // e.g., 0x7ffd...
printf("Value of p:   %p\n", p);    // same address
printf("Value at p:   %d\n", *p);   // 10 (dereferencing)

*p = 99;  // change value through pointer
printf("a is now: %d\n", a);  // 99
Variable a: [ 10 ] at address: 1000 Pointer p: [ 1000 ] at address: 2000 ↓ *p = 10 (value at address 1000)

Key Concepts

  • & — Address-of operator (gets address)
  • * — Dereference operator (gets value at address)
  • NULL pointer — pointer pointing to nothing: int *p = NULL;
  • Pointer arithmetic — p++ moves to next memory location of same type

Recursion & Factorial Program

Recursion is a technique where a function calls itself to solve smaller subproblems until a base case is reached.

  • Base Case — Condition that stops recursion (prevents infinite loop)
  • Recursive Case — Function calls itself with a smaller/simpler input
#include <stdio.h>

// Recursive factorial: n! = n × (n-1)!
int factorial(int n) {
    if (n == 0 || n == 1)   // Base Case
        return 1;
    return n * factorial(n - 1);  // Recursive Call
}

int main() {
    int num;
    printf("Enter a number: ");
    scanf("%d", &num);
    printf("%d! = %d\n", num, factorial(num));
    return 0;
}
// Input: 5  → Output: 5! = 120
factorial(4) = 4 × factorial(3) = 3 × factorial(2) = 2 × factorial(1) = 1 ← Base case = 2 × 1 = 2 = 3 × 2 = 6 = 4 × 6 = 24
Q 16

Structure in C

A structure is a user-defined data type that groups together variables of different data types under a single name.

Features

  • Groups related data of different types
  • Defined with struct keyword
  • Members accessed with . (dot) operator
  • Pointer to struct uses -> (arrow) operator
  • Can be nested (struct inside struct)
  • Can be passed to functions
#include <stdio.h>
#include <string.h>

// Define structure
struct Student {
    int   rollNo;
    char  name[50];
    float marks;
};

int main() {
    struct Student s1;   // Declare structure variable

    s1.rollNo = 101;
    strcpy(s1.name, "Vishw");
    s1.marks = 95.5;

    printf("Roll No : %d\n", s1.rollNo);
    printf("Name    : %s\n", s1.name);
    printf("Marks   : %.1f\n", s1.marks);

    // Pointer to structure
    struct Student *ptr = &s1;
    printf("Via ptr : %s\n", ptr->name);

    return 0;
}

Time Complexity & Space Complexity

Time Complexity

Measures how much time (number of operations) an algorithm takes relative to input size n. It does NOT measure actual clock time — it measures the growth rate of operations.

ComplexityNameExample
O(1)ConstantArray access arr[i]

Space Complexity

Measures the amount of memory an algorithm uses relative to input size. Includes: fixed space (variables) + variable space (input, dynamic memory, recursion stack).

  • Linear Search: Space O(1) — only a few variables
  • Merge Sort: Space O(n) — needs auxiliary array
  • Recursive factorial: Space O(n) — n frames on call stack

Binary Tree & Binary Search Tree (BST)

Binary Tree

A tree where each node has at most 2 children (left and right). No ordering constraint.

  • Root — topmost node
  • Leaf — node with no children

A binary tree with the property: Left subtree < Node < Right subtree for every node.

Q 27

Graph Traversal — BFS & DFS

A graph consists of vertices (nodes) and edges. Traversal means visiting all vertices systematically.

BFS — Breadth First Search

Explores graph level by level (layer by layer). Uses a Queue.

Graph: 1 — 2 — 5 | | 3 — 4 BFS from 1: Visit 1 → Visit 2,3 → Visit 4,5 Order: 1 → 2 → 3 → 4 → 5
  • Guarantees shortest path in unweighted graphs
  • Time: O(V + E), Space: O(V)
  • Applications: Shortest path, level-order tree traversal, social networks

DFS — Depth First Search

Goes as deep as possible along a branch before backtracking. Uses a Stack (or recursion).

Same Graph: DFS from 1: 1 → 2 → 4 → 3 → 5 (deep-first) (backtrack when stuck)
  • Good for detecting cycles, topological sort, maze solving
  • Time: O(V + E), Space: O(V)
FeatureBFSDFS
Data StructureQueueStack / Recursion
OrderLevel by levelDeep first
Shortest PathYes (unweighted)Not guaranteed
MemoryMore (stores level)Less
Q 28

Sorting Techniques: Bubble, Selection & Insertion Sort

1. Bubble Sort

Repeatedly compares adjacent elements and swaps them if out of order. Largest elements "bubble" to the end each pass.

// Bubble Sort
for(i=0; i<n-1; i++)
    for(j=0; j<n-i-1; j++)
        if(arr[j] > arr[j+1]) {
            temp=arr[j]; arr[j]=arr[j+1]; arr[j+1]=temp;
        }
// [64,25,12,22] → [25,12,22,64] → ... → [12,22,25,64]
  • Time: O(n²) worst/avg, O(n) best | Space: O(1)

2. Selection Sort

Finds the minimum element in the unsorted part and places it at the beginning.

for(i=0; i<n-1; i++) {
    minIdx = i;
    for(j=i+1; j<n; j++)
        if(arr[j] < arr[minIdx]) minIdx = j;
    // swap arr[i] and arr[minIdx]
    temp=arr[minIdx]; arr[minIdx]=arr[i]; arr[i]=temp;
}
// [64,25,12,22] → [12,25,64,22] → [12,22,64,25] → ...
  • Time: O(n²) all cases | Space: O(1) | Fewer swaps than Bubble

3. Insertion Sort

Builds a sorted array one element at a time by inserting each new element into its correct position.

for(i=1; i<n; i++) {
    key = arr[i];
    j = i - 1;
    while(j >= 0 && arr[j] > key) {
        arr[j+1] = arr[j];
        j--;
    }
    arr[j+1] = key;
}
// Like sorting playing cards in hand
  • Time: O(n²) worst, O(n) best (already sorted) | Space: O(1)

// Linear Search
for(i=0; i<n; i++)
    if(arr[i] == key) return i;  // found
return -1;  // not found

// Binary Search (on sorted array)
int low=0, high=n-1, mid;
while(low <= high) {
    mid = (low + high) / 2;
    if(arr[mid] == key) return mid;
    else if(arr[mid] < key) low = mid + 1;
    else                    high = mid - 1;
}
return -1;

Q 30

Hash Table & Collision Resolution Techniques

Hash Table

A hash table stores data as key-value pairs. A hash function maps keys to array indices (called buckets) for O(1) average-case access.

Hash Function:  index = key % tableSize
Example: key=25, tableSize=10 → index = 25 % 10 = 5
Store 25 at arr[5]

What is a Collision?

When two different keys map to the same index, it's called a collision. Example: key=15 and key=25 both map to index 5 (mod 10).

Collision Resolution Techniques
